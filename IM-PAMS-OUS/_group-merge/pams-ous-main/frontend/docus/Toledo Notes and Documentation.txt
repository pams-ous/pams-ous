# PUP OUS - Developer's Technical Guide

This document serves as a code-centric reference for the current frontend implementation.

## 1. Design System & CSS Variables (`css/style.css`)
We have extracted the mockup's visual identity into a set of global CSS variables. Use these to ensure consistency:

### Colors
- `--maroon: #800000;` (Primary Brand)
- `--gold: #DAA520;` (Secondary Accent)
- `--off-white: #f8f4f4;` (Page Background)
- `--text: #2d2d2d;` (Main Typography)

### Layout & Borders
- `--radius: 10px;` (Standardized for cards and large containers)
- `--shadow: 0 2px 12px rgba(0,0,0,.10);` (Light elevation)

## 2. Global Components (Reusable Classes)
Instead of writing new CSS for every element, utilize these established classes:

### Buttons (`.btn`)
- `.btn-primary`: Maroon background, white text.
- `.btn-gold`: Gold background, white text.
- `.btn-outline`: Maroon border, transparent background.
- *Usage:* `<button class="btn btn-primary">Text</button>`

### Cards (`.card`)
- Provides the white background, shadow, and padding seen in the mockup.
- Use `.card-header` and `.card-title` for internal structure.

### Badges (`.badge`)
- `.badge-pending`, `.badge-completed`, `.badge-urgent`, `.badge-admin`.
- *Usage:* `<span class="badge badge-completed">Done</span>`

### Form Controls (`.form-control`)
- Applied to `<input>`, `<select>`, and `<textarea>` for consistent padding, border-radius (8px), and maroon focus states.

## 3. Utility Classes
Extracted from the mockup to speed up development:
- `.flex`, `.flex-col`, `.items-center`, `.justify-between` (Flexbox shortcuts)
- `.mt-1` through `.mt-4` (Standardized margin-top increments)
- `.color-maroon`, `.color-gold`, `.color-gray` (Quick text coloring)
- `.fw-600`, `.fw-700` (Font weight helpers)

## 4. External Dependencies
These are preserved from the mockup via `@import` in `style.css`:
- **Font:** 'Poppins' (Weights: 300, 400, 500, 600, 700)
- **Icons:** Font Awesome 6.5.0

## 5. JavaScript Implementation (`js/script.js`)
- Uses `DOMContentLoaded` to ensure the DOM is ready.
- Logic is modular; check for element existence before attaching listeners (e.g., `if (loginBtn) { ... }`).
- Optimized for event delegation where possible.

## 6. Project Structure
- **/css/style.css**: Add global/reusable component styles here.
- **/js/script.js**: Main entry for UI logic.
- **/assets/**: Place all images (like the PUP Seal) here.
- **/mockup/**: Reference this for layout inspiration, but write all new production code in the root directory.
