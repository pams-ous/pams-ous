# PAMS-OUS: Developer & AI Agent Guidelines

This document serves as a high-priority context anchor for AI agents (Gemini, Claude, GPT, etc.) and developers working on the PAMS-OUS frontend. To maintain system integrity and readability, please adhere to the following standards established in May 2026.

## 1. JavaScript Architecture
*   **Centralized Config:** ALL environment-specific constants (API URLs, feature flags, versioning) MUST be stored in `js/config.js` via the global `CONFIG` object. Do not hardcode URLs in individual feature scripts.
*   **Modular Scripts:** Keep logic focused. `auth.js` handles all authentication forms. Page-specific logic should reside in dedicated files (e.g., `landing.js`).
*   **Mocking:** Use the `CONFIG.USE_MOCK_API` flag to toggle between prototype (timeout-based) and production (fetch-based) logic.

## 2. CSS & Styling Standards
*   **Variables First:** Use CSS variables (defined in `:root`) for colors, spacing, and transitions. Avoid hardcoded hex codes.
    *   Primary: `var(--maroon)`
    *   Secondary: `var(--gold)`
    *   Hover: `var(--maroon-hover)`, `var(--gold-hover)`
*   **Organization:** `style.css` is organized by: Variables -> Reset -> Utilities -> Components -> Page Layouts. Maintain this hierarchy.
*   **Transitions:** Use `var(--transition)` for all interactive elements to ensure a cohesive UI feel.

## 3. HTML & Accessibility (A11y)
*   **ARIA Labels:** Every form, input, and interactive button MUST have appropriate `aria-label`, `aria-required`, or `role` attributes.
*   **Semantic Links:** Use `<label for="ID">` for all form inputs.
*   **Professionalism:** Use professional placeholders (e.g., `personnel@pup.edu.ph`, `Juan Dela Cruz`). Do not use internal/humorous placeholders in tracked files.

## 4. Maintenance
*   **Redundancy:** Check for existing utility classes in `style.css` before creating new ones.
*   **Dead Code:** Before adding new scripts, ensure they are linked in the relevant HTML files. Remove any orphaned scripts or styles immediately.
